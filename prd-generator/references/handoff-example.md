# 交接示例：prd-generator -> prototype-generator

本示例展示 PRD 生成后，如何把可视化所需信息交给 `prototype-generator`。示例内容为虚构业务场景，只用于说明交接粒度。

## PRD 摘要

- **功能名称**：订单异常处理优化
- **建议原型类型**：桌面 Web，包含用户端订单详情和客服后台订单详情
- **原型目标**：让评审人员能看到异常状态如何展示、客服如何处理、标注如何映射到需求。
- **非原型目标**：不模拟真实支付、库存、履约接口；不实现真实权限系统；不追求高保真视觉。

## 页面清单

| 页面 ID | 页面名称 | 目标用户 | 页面目标 |
| --- | --- | --- | --- |
| PAGE-01 | 用户端订单详情 | 普通用户 | 展示异常原因、建议动作和联系客服入口 |
| PAGE-02 | 客服后台订单详情 | 一线客服/客服主管 | 展示异常来源、建议动作、处理记录和处理入口 |
| PAGE-03 | 异常处理弹窗 | 客服 | 二次确认并提交处理动作 |

## 需求 ID 到页面模块映射

| 需求 ID | 页面 | 模块 | 原型表现 |
| --- | --- | --- | --- |
| REQ-001 | PAGE-01, PAGE-02 | 异常状态标签 | 显示支付异常/库存异常/履约异常/未知异常 |
| REQ-002 | PAGE-01 | 用户端异常说明卡片 | 展示原因、预计处理时间、重新支付/联系客服 |
| REQ-003 | PAGE-02 | 客服异常处理区块 | 展示异常来源、异常码、建议动作和处理记录 |
| REQ-004 | PAGE-03 | 高风险动作二次确认弹窗 | 展示动作风险、确认按钮和取消按钮 |
| DATA-001 | PAGE-02 | 异常详情字段 | 展示异常码、来源系统、发生时间、最近更新时间 |
| STATE-001 | PAGE-01, PAGE-02 | 状态切换控件 | 用 mock 控件切换不同异常状态以便评审 |

## 关键交互

1. 页面导航
   - 左侧导航在用户端订单详情、客服后台订单详情之间切换。

2. 异常状态切换
   - 提供 mock 状态切换：支付异常、库存异常、履约异常、未知异常、已处理。
   - 切换后更新用户端文案和客服端建议动作。

3. 用户端操作
   - 支付异常时展示“重新支付”按钮。
   - 未知异常时展示“联系客服”按钮。

4. 客服处理动作
   - 点击“提交处理动作”打开二次确认弹窗。
   - 提交成功后追加一条处理记录，并显示 Toast。

5. 标注联动
   - 点击异常状态标签，右侧显示 `REQ-001`。
   - 点击异常说明卡片，右侧显示 `REQ-002`。
   - 搜索 `REQ-003` 时切换到客服页面并高亮处理区块。

## 状态与异常

| 状态 ID | 状态 | 需要可视化 |
| --- | --- | --- |
| STATE-001 | 支付异常 | 是 |
| STATE-002 | 库存异常 | 是 |
| STATE-003 | 履约异常 | 是 |
| STATE-004 | 未知异常 | 是 |
| STATE-005 | 已处理 | 是 |
| STATE-006 | 上游异常码缺失 | 非可视化需求，在标注说明 |

## 非可视化需求

| 需求 ID | 内容 | 说明 |
| --- | --- | --- |
| DATA-002 | 异常处理记录需要写入审计日志 | 后端/测试关注，不直接出现在用户 UI |
| METRIC-001 | 上报 order_exception_view | 埋点需求，在标注中说明触发时机 |
| NFR-001 | 高风险动作需要权限校验 | 权限逻辑在标注中说明，不实现真实权限系统 |

## Prototype Generator Handoff

- **原型范围**：3 个页面/视图，用户端订单详情、客服后台订单详情、异常处理弹窗。
- **默认设备**：桌面 Web。
- **核心模块**：异常状态标签、用户异常说明卡片、客服异常处理区块、处理记录列表、二次确认弹窗、右侧标注面板。
- **需求 ID 规则**：沿用 PRD ID；新增状态使用 `STATE-00x`；非可视化需求使用原 PRD 类型 ID。
- **关键交互**：页面切换、状态切换、弹窗、提交反馈、标注搜索和高亮。
- **假设**：使用 mock 数据；不连接真实后端；客服主管权限用视图切换或标注说明代替。
- **待确认问题**：客服高风险动作的具体列表；预计处理时间文案；未知异常兜底文案。
