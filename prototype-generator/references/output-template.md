# HTML 原型输出规范

默认生成单文件 HTML，便于预览、分享和归档。只有用户明确要求或项目需要时，再拆成多文件。

## 单文件输出

默认文件名：

```text
prototype.html
```

单文件应包含：

- HTML 结构
- CSS 样式
- JavaScript 交互
- Mock 数据
- 标注数据
- 页面切换逻辑
- 标注搜索和高亮逻辑

## 页面结构建议

```html
<body>
  <div class="app-shell">
    <aside class="sidebar">导航</aside>
    <main class="prototype-stage">原型页面</main>
    <aside class="annotation-panel">标注面板</aside>
  </div>
</body>
```

默认布局：

- 左侧：页面导航或流程导航
- 中间：原型页面
- 右侧：标注说明

移动端 PRD 可以改为：

- 中间：手机尺寸画布，默认 `375px x 812px`
- 右侧：标注面板
- 顶部：页面/角色切换
- 大屏手机可使用 `390px x 844px`
- 在桌面评审页面中居中展示手机画布，不要把移动端 UI 拉满浏览器宽度
- 手机画布外层可使用轻量边框和阴影，但不要影响需求阅读

## CSS 要求

- 使用稳定尺寸和响应式约束，避免交互时布局跳动。
- 信息密度适中，适合产品评审。
- 避免过度装饰、营销页风格、大面积渐变和无意义插图。
- 按钮、输入框、表格、Tab、弹窗、Toast、状态标签要清晰可辨。
- 文本不能溢出或互相遮挡。
- 标注高亮要明显但不干扰阅读。

## Chat 输出限制

当原型超过 3 个页面、交互状态较多，或标注数据较大时：

- 不要在 chat 中输出完整 HTML。
- 建议写入 `prototype.html` 或多文件结构。
- chat 中只输出原型范围、页面结构、关键交互、标注使用方式和文件路径。
- 如果用户明确要求完整 HTML，再输出完整内容。

## JavaScript 要求

至少支持：

- 页面导航
- 模块点击选中
- 标注面板更新
- 标注搜索
- 搜索结果定位并高亮模块
- 弹窗打开/关闭
- Tab 切换
- 简单表单校验
- 状态切换或 mock 操作反馈

所有交互应在本地浏览器中运行，不依赖网络。

## 标注要求

每个关键模块应包含：

- `data-annotation-id`
- `data-module-id`
- 可点击区域
- 对应标注数据

标注面板至少展示：

- 需求 ID
- 页面/模块
- 功能说明
- 交互规则
- 状态说明
- 验收标准
- 待确认问题

## 多文件输出

用户明确要求多文件时：

```text
index.html
styles.css
app.js
annotations.json
```

多文件输出应保持职责清晰：

- `index.html`：结构
- `styles.css`：样式
- `app.js`：交互
- `annotations.json`：标注数据

## 输出前说明

最终交付时，简要说明：

- 生成了哪些页面
- 支持哪些交互
- 标注如何使用
- 哪些需求未可视化
- 哪些内容是假设或待确认
