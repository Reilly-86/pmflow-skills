# 交接示例：prototype-generator -> 评审/协作

本示例展示 HTML 原型生成后，如何把原型交给产品、研发、测试、设计和运营进行评审。示例内容为虚构业务场景，只用于说明交接粒度。

## 原型交付摘要

- **原型名称**：订单异常处理优化交互原型
- **输出形式**：单文件 `prototype.html`
- **原型类型**：桌面 Web 评审级原型
- **覆盖页面**：
  - 用户端订单详情
  - 客服后台订单详情
  - 异常处理确认弹窗
- **主要目标**：验证异常状态展示、客服处理动作和 PRD 标注是否清楚。

## 需求覆盖摘要

| 需求 ID | 原型位置 | 覆盖方式 | 状态 |
| --- | --- | --- | --- |
| REQ-001 | 异常状态标签 | 可视化 | 已覆盖 |
| REQ-002 | 用户端异常说明卡片 | 可视化 | 已覆盖 |
| REQ-003 | 客服异常处理区块 | 可视化 | 已覆盖 |
| REQ-004 | 二次确认弹窗 | 可视化 | 已覆盖 |
| DATA-002 | 审计日志写入 | 非可视化标注 | 已标注 |
| METRIC-001 | order_exception_view 埋点 | 非可视化标注 | 已标注 |

## 可评审交互

1. 点击左侧导航切换用户端和客服端页面。
2. 使用状态切换控件查看支付异常、库存异常、履约异常、未知异常和已处理状态。
3. 点击用户端异常说明卡片，右侧标注显示 `REQ-002`。
4. 在右侧搜索框输入 `REQ-003`，原型切换到客服页面并高亮异常处理区块。
5. 点击“提交处理动作”，查看二次确认弹窗和提交成功反馈。

## 标注使用说明

- 页面中带高亮提示的模块都绑定了需求 ID。
- 点击模块可在右侧查看功能说明、交互规则、状态说明和验收标准。
- 点击右侧标注结果会定位到对应页面模块。
- “非可视化需求”用于展示后端、数据、埋点、权限和审计类要求。

## 评审关注点

### 产品

- 异常状态分类是否符合业务语义。
- 用户端文案是否能降低咨询。
- P0/P1/P2 范围是否正确。

### 研发

- 异常码映射是否可实现。
- 客服处理动作是否需要异步状态。
- 审计日志和处理记录的数据结构是否明确。

### 测试

- 状态切换和异常兜底是否覆盖。
- 重复提交、权限不足、处理失败是否需要补充用例。
- 非可视化需求是否都有可验证方式。

### 设计

- 用户端异常提示层级是否清楚。
- 客服后台信息密度是否合适。
- 弹窗确认文案是否足够明确。

### 运营/客服

- 预计处理时间和建议动作是否符合实际处理流程。
- 客服动作是否需要 SOP 或权限审批。

## 待确认问题

| 问题 | 影响 | 建议负责人 |
| --- | --- | --- |
| 未知异常的用户端文案是否使用统一兜底 | 影响用户端展示 | 产品/客服 |
| 高风险动作是否都需要主管权限 | 影响弹窗和后台权限 | 客服负责人/研发 |
| 异常处理记录是否对用户可见 | 影响用户端页面结构 | 产品 |

## 后续动作

1. 产品根据评审意见更新 PRD。
2. 原型根据 PRD 变更同步调整页面模块和标注。
3. 研发基于需求 ID 和标注拆分开发任务。
4. 测试基于状态和非可视化需求补充测试用例。
